# 第2章 为什么我们要“离散化”？

在这一章里，我们要解决一个看起来很奇怪的问题：为什么很多算法题里数值越大，反而越容易做？

## 1 一个简单的问题
给你一个数组：
`A = [1, 3, 2, 4, 5]`

**问题**：有多少个逆序对？（定义：`i < j` 且 `A[i] > A[j]`）

思考：如果数组范围是 `1 ≤ A[i] ≤ 10`，你会怎么做？

很多人会想到用一个频率数组 `cnt[x]` 来记录每个值出现的次数，然后从后往前遍历，对于每个 `A[i]`，查询比它小的数出现了多少个，同时更新 `cnt[A[i]]`。这样可以快速计算逆序对。

## 2 问题突然变难
现在题目稍微改变。数组变为：
`A = [1000000000, 7, 500000000, 3]`

范围是 `1 ≤ A[i] ≤ 10^9`。

**问题**：如果你还想用 `cnt[x]` 数组，它需要多大？

答案是 `10^9`，这显然会导致内存爆炸。即使你使用哈希表，也需要存储大量键值对，而且无法高效地进行范围查询。

**问题 1**：我们真的需要关心数字的真实大小吗？例如，`1000000000` 这个值真的重要吗？

## 3 再看一眼问题本质
逆序对的判断条件是 `A[i] > A[j]`。注意：**我们只关心大小关系，而不是具体数值。**

举例：
原数组 `[1000000000, 7, 500000000, 3]` 其实只需要知道：
`1000000000 > 500000000 > 7 > 3`

如果按大小排序后重新编号：
`3 → 1`
`7 → 2`
`500000000 → 3`
`1000000000 → 4`

原数组就变成了 `[4, 2, 3, 1]`。

**关键问题**：新数组 `[4, 2, 3, 1]` 和原数组的逆序对数量会改变吗？

**答案**：不会。因为大小关系被完全保留了下来。在新数组中，`4>2`、`4>3`、`4>1`、`2>1`、`3>1` 对应着原数组中 `1000000000>7`、`1000000000>500000000`、`1000000000>3`、`7>3`、`500000000>3` 这些关系。

## 4 离散化出现了
于是我们得到一个新思想：

**问题 2**：如果我们把所有数值重新编号成 `1, 2, 3, ..., n`，是不是就可以用数组解决问题了？

**答案**：是的。这个过程就叫**离散化**。我们将巨大的、稀疏的值域，映射到紧凑的、连续的下标上。

## 5 离散化到底是什么？
离散化就是把原来可能很大的数值，映射到一个连续的小范围内，同时保持大小关系不变。

形式化地说：
- 原数组 `A[i]`，值域范围可能很大（如 `0 ~ 10^9`）。
- 离散化后，我们得到 `rank(A[i])`，值域是紧凑的 `1 ~ m`（`m` 是不同数字的个数，`m ≤ n`）。

## 6 如何实现离散化？
**问题 3**：如何计算一个值 `x` 的排名 `rank(x)`？

标准步骤如下：

1.  **收集**：将所有需要离散化的值放入一个数组 `tmp` 中。
2.  **排序**：对 `tmp` 进行升序排序。例如 `tmp = [3, 7, 500000000, 1000000000]`。
3.  **去重**：移除 `tmp` 中重复的元素（这是关键的一步，确保每个值映射到唯一的排名）。
4.  **映射**：对于任意值 `x`，其排名 `rank(x)` 等于 `x` 在去重后的 `tmp` 数组中的下标（通常从1开始）。

具体代码实现（C++风格伪代码）：
```cpp
// 假设原数组为 a，长度为 n
vector<int> tmp(a, a + n); // 1. 收集
sort(tmp.begin(), tmp.end()); // 2. 排序
tmp.erase(unique(tmp.begin(), tmp.end()), tmp.end()); // 3. 去重
// 4. 映射函数
int getRank(int x) {
    return lower_bound(tmp.begin(), tmp.end(), x) - tmp.begin() + 1;
}
```

## 7 为什么必须去重？
思考：数组 `[5,5,5,5]` 如果不去重，数字 `5` 的排名会变得混乱。不去重时，`tmp = [5,5,5,5]`，`5` 的下标可能是 `0,1,2,3`，不唯一。去重后，`tmp = [5]`，`5` 的排名始终是 `1`。所有相同的元素都映射到同一个值，保证了映射的一致性，这是正确的，因为逆序对判断中相等的数不算逆序。

## 8 离散化真正解决的问题
很多算法（如树状数组、线段树、差分、扫描线）的核心数据结构需要使用数组下标。但当题目给出的坐标或数值范围极大（如 `10^9`）时，无法直接开辟那么大的数组。此时，就必须通过离散化，将巨大的、稀疏的“坐标”压缩成紧凑的、连续的“索引”，从而让这些数据结构得以应用。

**例如**：我们用树状数组求逆序对。树状数组的下标范围需要覆盖所有可能的值。如果值域是 `10^9`，树状数组开不了那么大。离散化后，我们将值域压缩到 `1~n`，就可以用大小为 `n` 的树状数组解决了。

## 9 一个经典应用场景：区间覆盖统计
给你 `n` 个区间 `[l, r]`，例如 `[1, 1000000000]`，`[5,7]`，`[8,9]`。问某个点被覆盖了多少次。

**暴力思路**：开一个数组 `cover[1..10^9]`，对每个区间 `[l, r]` 执行 `cover[l..r]++`，最后查点。这显然不可能。

**离散化做法**：
1.  收集所有区间的端点：`{1, 1000000000, 5, 7, 8, 9}`。
2.  排序去重：`{1, 5, 7, 8, 9, 1000000000}`。
3.  将这些端点映射到 `1~6`。
4.  原来的区间操作 `[1, 1000000000]` 转化为对映射后下标 `[1, 6]` 的操作（在差分数组上）。
5.  查询时，先将查询点映射到对应下标，再查值。

这样，问题的规模瞬间从 `10^9` 量级缩小到 `2n` 量级。

## 10 离散化的本质
很多人觉得离散化是技巧。其实不是。它的本质是：
**我们只关心“相对关系”（如大小、顺序、区间端点之间的相对位置），而不关心“绝对值”。**
因此，数值可以被安全地重新编号，而不影响问题的解。这体现了算法设计中的一个重要思想：**保留关键信息，忽略不必要细节，从而降低问题复杂度**。

## 11 真正理解离散化（思考题）
如果你已经理解，你应该能回答：

1.  **问题 1**：为什么离散化常常和扫描线算法一起出现？
    > 因为扫描线算法通常处理平面上的矩形，矩形的 `x` 和 `y` 坐标范围可能很大。但矩形的数量有限，我们只关心矩形的边界线（`x` 坐标）和扫描线与矩形边的交点（`y` 坐标）。将 `y` 坐标离散化后，就可以用线段树等数据结构高效维护被覆盖的区间状态。

2.  **问题 2**：为什么线段树在解决许多区间问题时，经常必须先进行离散化？
    > 线段树通常需要建立在线性的下标上。如果问题的坐标（如位置、数值）范围非常大但实际出现的点很少，直接建树会导致树非常大，内存和时间都无法承受。离散化将这些稀疏的坐标压缩为连续的下标，使得线段树的大小与实际问题规模（点数）成正比，从而变得可行。

3.  **问题 3**：在二维平面扫描线问题中，为什么通常只需要离散化其中一个坐标（例如 `y` 坐标）？
    > 在经典的矩形面积并问题中，我们沿 `x` 轴扫描。对于每个 `x` 位置，我们需要知道当前扫描线上被矩形覆盖的 `y` 轴区间总长度。这些区间的端点只来自矩形的上下边，数量是 `O(n)` 的。离散化 `y` 坐标后，我们可以将连续的 `y` 轴区间表示为离散化的区间段，用线段树维护每个区间段是否被覆盖。而 `x` 坐标虽然也可能很大，但我们只需要在矩形的左右边界处（事件点）处理，不需要为所有 `x` 值建立结构，所以通常不需要离散化 `x` 坐标。
